2. Validation Methodology

Pipeline

Produce Tagged PDF → veraPDF --flavour ua1 → Parse Findings → Report

Pre-veraPDF Font Audit

Before invoking veraPDF, we run an 8-code font audit that catches approximately 90% of font-related failures before they reach the validator. This pre-flight step identifies:

  • Missing or invalid ToUnicode CMaps
  • Standard 14 fonts lacking embedded programs
  • Broken CIDSet descriptors
  • Width mismatches between font descriptor and actual glyph widths
  • Missing Unicode mappings for glyph IDs
  • Type3 fonts (flagged as non-repairable)
  • Encoding inconsistencies across composite fonts
  • CIDToGIDMap validation failures

veraPDF as Authority

veraPDF formalizes every machine-verifiable requirement from PDF/UA-1 into executable rules. Each rule is mapped to a specific clause in ISO 14289-1:2014, enabling automated conformance checking against the full specification.[3]

Validator Disagreement

Caveat: Validator Inconsistency

  • A study of 155 PDF files revealed leading validators produce inconsistent results in 50.3% of cases.[6]
  • We use veraPDF as our primary authority but acknowledge that validator disagreement exists across the industry.
  • When possible, we validate against multiple tools and investigate discrepancies.

3.6 Reading Order

Line-group epsilon (6pt) fixes right-to-left bugs caused by sub-pixel y-coordinate jitter across text fragments on the same visual line.

Tested on multi-column legal filings with court header stamps. Example correction:

Before (broken)After (fixed)
"DECISION ORDER" / "PAUL RUGGIERO" / "&""PAUL RUGGIERO" / "DECISION ORDER" / "&" — correct LTR

Fragments within the epsilon threshold are grouped and sorted left-to-right, eliminating false reading-order inversions in headers, stamps, and multi-line title blocks.

4. Known Limitations

Limitations and Caveats

  • Heuristic, not ML. Auto-tagging relies on structural heuristics, not machine learning. Accuracy depends on how well the source document follows standard formatting conventions.
  • Tables without signals. Tables lacking any structural cues (no gridlines, no column alignment, no whitespace patterns) may not be detected.
  • Type3 fonts. Type3 fonts cannot be made accessible. They are flagged in the font audit but cannot be automatically repaired.
  • Complex floating layouts. Reading order for documents with complex multi-column layouts and floating elements (sidebars, pull-quotes, wrapped figures) may require manual review.
  • Native mode is new. Our native tagging mode should be verified with --mode auto, which provides per-page fallback to raster when operator matching confidence is below threshold.
  • Validator disagreement. veraPDF and other validators (PAC, Acrobat Preflight) disagree on certain rules.[6] We target veraPDF UA1 compliance as our primary conformance benchmark.
